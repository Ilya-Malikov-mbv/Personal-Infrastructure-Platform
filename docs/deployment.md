# Развертывание

## Назначение

В данном документе описан процесс развертывания инфраструктуры на чистом VPS.

Документ предполагает наличие:

- Ubuntu Server 24.04 LTS
- доменного имени
- пользователя с правами sudo
- SSH-доступа к серверу

---

# Итоговая структура

После завершения развертывания будут доступны следующие сервисы.

| Сервис        | Адрес                      |
| ------------- | -------------------------- |
| Основной сайт | https://mal-mbv.ru         |
| Grafana       | https://grafana.mal-mbv.ru |
| Uptime Kuma   | https://status.mal-mbv.ru  |

---

# Этапы развертывания

## 1. Подготовка сервера

Установка базовых пакетов.

```bash
sudo apt update
sudo apt upgrade -y
```

---

## 2. Установка Docker

Устанавливается Docker Engine и Docker Compose.

После установки проверяется:

```bash
docker --version
docker compose version
```

---

## 3. Настройка Nginx

Используется один экземпляр Nginx, который выполняет функции:

- обслуживания статического сайта;
- reverse proxy;
- HTTPS;
- маршрутизации между сервисами.

После изменения конфигурации обязательно выполняется проверка.

```bash
sudo nginx -t
sudo systemctl reload nginx
```

---

## 4. Получение TLS-сертификатов

HTTPS обеспечивается сертификатами Let's Encrypt.

Для каждого домена выпускается отдельный сертификат.

После выпуска производится автоматическое продление Certbot.

---

## 5. Развертывание сайта

Статические файлы размещаются в директории веб-сервера.

После изменения содержимого сайт становится доступен без дополнительных действий.

---

## 6. Развертывание мониторинга

Через Docker Compose запускаются:

- Prometheus
- Grafana
- Uptime Kuma

Запуск производится одной командой.

```bash
docker compose up -d
```

---

## 7. Настройка Prometheus

Prometheus получает системные метрики от Node Exporter.

Проверка состояния:

```bash
curl http://127.0.0.1:9090/-/healthy
```

---

## 8. Настройка Grafana

После запуска:

- добавляется источник данных Prometheus;
- импортируются дашборды;
- проверяется получение метрик.

---

## 9. Настройка Uptime Kuma

Создаются мониторы:

- основной сайт;
- Grafana;
- собственный интерфейс Uptime Kuma.

После этого сервис начинает автоматически контролировать доступность инфраструктуры.

---

## 10. Настройка GitHub Actions

После каждого Push в основную ветку выполняется:

1. SSH-подключение к серверу.
2. Обновление репозитория.
3. При необходимости перезапуск сервисов.

Это позволяет автоматически разворачивать изменения без ручного подключения к VPS.

---

# Проверка работоспособности

После завершения настройки рекомендуется проверить:

- сайт открывается по HTTPS;
- Grafana отображает метрики;
- Uptime Kuma показывает все сервисы в статусе Up;
- сертификаты действительны;
- Docker-контейнеры находятся в состоянии Up.

Проверка контейнеров:

```bash
docker ps
```

Проверка Nginx:

```bash
sudo systemctl status nginx
```

Проверка открытых портов:

```bash
sudo ss -tlnp
```

# Обновление инфраструктуры

Обновление Docker-контейнеров:

```bash
docker compose pull
docker compose up -d
```

Обновление проекта:

```bash
git pull
```

После изменения конфигурации Nginx:

```bash
sudo nginx -t
sudo systemctl reload nginx
```
