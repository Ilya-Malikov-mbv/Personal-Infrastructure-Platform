# Docker

## Назначение

В данной директории находится конфигурация контейнеризированных сервисов инфраструктуры.

Для управления используется Docker Compose.

---

## Состав сервисов

| Сервис      | Назначение                      |
| ----------- | ------------------------------- |
| Prometheus  | Сбор системных метрик           |
| Grafana     | Визуализация метрик             |
| Uptime Kuma | Мониторинг доступности сервисов |

---

## Структура

```text
docker/
└── docker-compose.yml
```

---

## Запуск

Запуск всех сервисов:

```bash
docker compose up -d
```

Просмотр работающих контейнеров:

```bash
docker ps
```

Остановка:

```bash
docker compose down
```

Перезапуск:

```bash
docker compose restart
```

---

## Обновление контейнеров

```bash
docker compose pull

docker compose up -d
```

---

## Хранение данных

Используются именованные Docker Volumes.

Это позволяет сохранять данные между обновлениями контейнеров.

Например:

- данные Grafana;
- конфигурация Uptime Kuma.

---

## Сетевое взаимодействие

Внешний доступ к контейнерам отсутствует.

Публикуются только локальные порты:

| Сервис      | Порт           |
| ----------- | -------------- |
| Prometheus  | 127.0.0.1:9090 |
| Grafana     | 127.0.0.1:3000 |
| Uptime Kuma | 127.0.0.1:3001 |

Доступ пользователей осуществляется исключительно через Nginx Reverse Proxy.
